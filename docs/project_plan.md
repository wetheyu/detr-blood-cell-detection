# 项目计划

## 项目名称

基于 DETR 的血细胞目标检测系统

## 建设目标

本项目将 DETR 迁移到血细胞目标检测场景，形成可训练、可评估、可复现、可演示的完整目标检测工程。交付内容包括：

- 数据集说明：来源、格式、类别、训练/验证划分、样本数量、标注统计。
- 模型方法：DETR 端到端检测流程、ResNet-50 Backbone、Transformer 编码解码结构、匈牙利匹配损失。
- 工程实现：训练、评估、批量预测、Web 演示入口清晰可运行。
- 实验设计：基线实验、学习率调参、查询数量调参和对比分析。
- 结果分析：训练损失曲线、验证集 AP/AP50/AP75、不同设置对比表、典型检测可视化。
- 复现资料：环境配置、运行命令、实验报告、指标表格和图表。

## 当前项目基础

- 数据格式：COCO detection。
- 类别数量：4 类，`cells`、`Platelets`、`RBC`、`WBC`。
- 训练集：765 张图片，10,342 条标注。
- 验证集：73 张图片，967 条标注。
- 最优模型：`experiments/runs/lr_low/checkpoint_best.pth`。
- 历史模型：`output_2/checkpoint.pth`。
- 训练日志：`output_2/log.txt` 和 `experiments/runs/*/log.txt`。

## 说明重点

1. DETR 将目标检测建模为集合预测问题，省去了传统检测器中的候选框生成和 NMS 后处理。
2. 本项目使用 COCO detection 格式组织血细胞数据，便于复用 DETR 原生训练和评估流程。
3. 实验包含基线、学习率调参和 object queries 数量调参，能够展示关键超参数对检测性能的影响。
4. 当前最优模型为 `lr_low`，验证集 AP=0.6365，AP50=0.9176，AP75=0.7455。
5. 数据规模相对较小，类别分布不均衡，小目标和密集目标仍是主要误差来源。
